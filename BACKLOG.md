#### 배치

#### 인물 캐스팅 (비유 일관성용)

| 역할 | 비유 |
|---|---|
| 신입 | 사전학습 끝난 LLM (책만 많이 읽은 상태) |
| 사수 김과장 | SFT 단계의 정답 예시 제공자 |
| 인사팀 | RM 학습을 위한 평가자 |

(기존 Tab9의 "부장님/김대리" 는 별개 맥락이라 재활용하지 않음)

#### 스텝 구조 (전환부 + 4 STEP 개념 + 1 STEP 심화 + 퀴즈)

**전환부 — "사전학습만으로는 부족한 이유"**
- 비유: "신입이 도서관에서 책만 읽으면 실무 못함"
- 1~2 문단, 스텝 번호 없이 도입만

**STEP 1 — SFT (지도 미세조정)**
- 제목: "사수 김과장이 '이럴 땐 이렇게 답해' 라고 시범 보여주기"
- 핵심 개념
  - 사전학습 모델 + 사람이 만든 정답 Q&A (수만 개)로 추가 학습
  - 일반 다음-토큰 예측(cross-entropy)과 동일한 손실함수, 데이터만 고품질
- 수식은 STEP 5 "심화: 수식 꼬리물기" 로 이동 (본 STEP은 개념 중심)
- 인터랙션
  - 예시 Q&A 하나 카드로 보여주고 "모델이 뭘 학습하는지" 하이라이트 (토큰 단위 기여도 시각화)
  - 또는 간단히: 정답/오답 토큰 대비 시각화 (정답은 파란색, 오답은 회색)
- 수식 표기: 기존 Tab6 (Transformer) 스타일 따르기 (ℝ, 벡터 표기)

**STEP 2 — 보상 모델 (Reward Model, RM)**
- 제목: "인사팀이 '이 답변이 더 낫다' 3,000번 고르기 → 평가표 학습"
- 핵심 개념
  - 동일 질문에 대한 응답 2개(A, B) → 사람이 선호 선택
  - Bradley-Terry 모델로 선호를 확률로 학습 (구체 수식은 STEP 5 심화 참조)
  - RM은 응답을 입력받아 스칼라 점수를 뱉는 함수가 됨
- 도식
  ```
  질문 Q ─┬─→ 모델 → 응답 A ─┐
          └─→ 모델 → 응답 B ─┘
                             ↓
                      사람이 A vs B 선택
                             ↓
                       RM 학습 데이터
  ```
- 인터랙션
  - 사용자에게 Q + 응답 2개 제시 → 어느 쪽 선호하는지 고르기 (3문제 정도)
  - 고른 결과 → RM이 그 패턴 학습하는 모습 설명

**STEP 3 — PPO (근접정책최적화)**
- 제목: "조심스럽게 자기 개선하기 — 보상은 높이되 너무 급변 X"
- 핵심 개념
  - SFT 모델이 이제 새 응답을 생성 → RM이 점수 매김 → 점수 높은 방향으로 업데이트
  - "근접"의 의미: 원래 SFT 모델에서 **너무 멀리 벗어나지 않도록** KL 페널티 추가
  - 왜? KL 없으면 reward hacking (RM만 높이고 말이 이상해짐)
- 수식은 STEP 5 심화 참조 (본 STEP은 "조심성" 슬라이더 체험 중심)
- 인터랙션
  - **"조심성" 슬라이더** (0 ~ 1) — 메인 UI에는 β·KL 같은 기호 표기 금지, 한글 비유 레이블만
  - 조심성 낮음: 보상 ↑ 그러나 말이 이상해짐 (시뮬레이션: RM 점수 높은데 말은 어색)
  - 조심성 높음: 안정적이지만 개선 느림
  - 슬라이더 움직이면 "이상적 균형점" 시각 피드백
  - STEP 5 수식 섹션에서 "이 '조심성'이 β·KL 페널티였구나" 연결되는 aha 구조

**STEP 4 — 전체 파이프라인 한 장 요약**
- 도식: 사전학습 → SFT → RM → PPO → 배포된 모델
- "사전학습 = 도서관에서 책 읽기" / "사후학습 = 사수+인사팀+승진 훈련"
- 각 단계의 데이터 규모·목적·비용 비교 카드
  | 단계 | 데이터 | 목적 | 비용 |
  |---|---|---|---|
  | 사전학습 | 인터넷 텍스트 ~1T 토큰 | 언어 능력 | 높음 |
  | SFT | 고품질 Q&A ~10만 개 | 답변 스타일 | 낮음 |
  | RM | 선호 비교 ~10만 쌍 | 평가 기준 | 중간 |
  | PPO | 생성-평가 루프 | 정책 미세조정 | 중간 |
  (수치는 대략치 — GPT-3.5 수준 공개 자료 기준)

**STEP 5 — 심화: 수식 꼬리물기 (옵셔널)**
- 의도: 개념은 STEP 1~4 에서 이미 이해했으니, 여기서는 **궁금한 사람만 파고드는 공간**
- UX: **중첩(꼬리물기) 접이식 패턴** — 펼치면 또 펼칠 게 나옴
  ```
  [수식으로 깊이 보기]  ← 최상위 토글 (기본 접힘)
    ↓ 펼치면
    [1. SFT — 다음 토큰 예측]  ← 2단계 토글
      ↓ loss = −log p(정답 토큰 | 입력)
      [유도 과정 보기]  ← 3단계 토글
        ↓ cross-entropy loss 유도, 왜 −log 인지, 확률의 의미
    [2. RM — Bradley-Terry]
      ↓ P(A > B) = σ(r_A − r_B)
      ↓ loss_RM = −log σ(r(선호) − r(비선호))
      [유도 과정 보기]
        ↓ 왜 두 응답 점수 차이가 선호 확률이 되는지 (sigmoid 해석)
    [3. PPO — 보상 + KL 페널티]
      ↓ 목표 = E[r(응답)] − β · KL(π_new ∥ π_SFT)
      [KL divergence란?]  ← 4단계 토글
        ↓ 두 분포의 "다름" 측정, 왜 이게 "너무 멀어지지 마"인지
      [β의 역할 — "조심성" 슬라이더의 정체]
        ↓ STEP 3에서 "조심성"으로 체험한 그 값이 바로 β. 수식 관점에서 복습
  ```
- 스타일: 모바일에서도 잘 동작하는 얕은 계층 우선 (각 토글은 1번 탭으로 즉시 펼침)
- 각 수식 옆에 "**LaTeX 원본 보기**" 토글로 `$loss = -\log p(y|x)$` 도 선택적으로
- 이 STEP은 **퀴즈에 반영하지 않음** (원본 요청 "진도율 미반영" 준수)

**STEP 6 — 퀴즈 3~5문항**
- 기존 Tab9/Tab6의 `QuickQuiz` 또는 유사 컴포넌트 재활용
- **STEP 1~4 개념만 평가** (수식은 심화 STEP이라 퀴즈에 포함 X)
- 예시 문항
  1. "SFT의 학습 데이터는 주로 누가 만드나요?" (객관식: 사람 라벨러 / 크롤링 / 모델 자체생성)
  2. "RM의 역할은?" (점수 함수 / 생성 모델 / 분류기 중)
  3. "PPO에서 KL 페널티가 없으면 생기는 문제는?" (보상 해킹 / 학습 안 됨 / 느려짐 중)
